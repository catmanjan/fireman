PROJECT FIREMAN

A Linux server-side firewall management application.

Project Fireman

Firewall management is difficult. Using common tools such as iptables, while effective, often result in many lines of rules which can be painful to manage and inefficient to run. This may be due to the fact that users will often write firewall rules in an arbitrary order and avoid modifying rules due to time constraints. This complexity can impair the security efforts of an organisation.

The goal of this project is to develop simple and easy to use server side firewall configuration tools. There would be two separate tools - the first focusing on ease of use and the second focusing on increased performance.

The complexity in firewall rule management can result in misconfigured service-specific firewall rules, leaving the default policy of accepting all packets. This is commonplace in the industry. In principle, firewall rules should be as easy to manage as the network services they are meant to protect. We aim to develop a tool that can summarise and represent firewall rules in a human readable form, reducing complexity, and the effort required to manage them.
